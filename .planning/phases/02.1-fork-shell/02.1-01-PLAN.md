<task type="auto">
  <name>Task 1: Add caelestia-shell flake input and wire homeManagerModules.default</name>
  <files>flake.nix, flake.lock</files>
  <action>
Edit flake.nix to add caelestia-shell as a pinned input and feed its home-manager module into home-manager.sharedModules.

1. In `inputs`, add AFTER the stylix block:
```nix
caelestia-shell = {
  url = "github:caelestia-dots/shell";
  inputs.nixpkgs.follows = "nixpkgs";
};
```
Follow nixpkgs (per project convention — all inputs currently do this). Do NOT follow home-manager; caelestia's flake does not take home-manager as an input.

2. In `outputs = { self, nixpkgs, home-manager, stylix, ... }:` add `caelestia-shell` to the destructured args:
```nix
outputs = { self, nixpkgs, home-manager, stylix, caelestia-shell, ... }:
```

3. Inside the home-manager module block (currently lines 29-33), add `sharedModules` BEFORE the users line:
```nix
{
  home-manager.useGlobalPkgs = true;
  home-manager.useUserPackages = true;
  home-manager.sharedModules = [ caelestia-shell.homeManagerModules.default ];
  home-manager.users.pilot = import ./modules/home;
}
```

4. Run `nix flake lock --update-input caelestia-shell` (or `nix flake lock` if the input does not yet exist in flake.lock) to pin the current HEAD into flake.lock. If the command fails due to network/sandbox, run `nix flake update` instead.

5. Run `nix flake check --no-build` to validate eval. If it errors, read the error and fix — do NOT skip.

DO NOT add `pkgs.quickshell` to environment.systemPackages anywhere. Caelestia's flake pulls quickshell transitively; adding it manually creates version skew (Research Pitfall 1).

**Step A — Create modules/home/caelestia.nix** (new file):

```nix
{ ... }:
{
  # Caelestia shell: unified QuickShell bar + launcher + notifications + lock + sidebar.
  # Replaces the hand-rolled waybar/rofi/mako/hyprlock/hyprpaper/cliphist stack.
  #
  # Theming note: Stylix does NOT target caelestia. Stylix still owns kitty,
  # GTK, Qt, cursor, and the system font stack. Caelestia themes its own QML
  # surfaces via programs.caelestia.settings. Do NOT try to bridge them here;
  # pilot-specific color injection is a later phase (matugen + specialisations).
  programs.caelestia = {
    enable = true;

    # Start via systemd user service bound to graphical-session.target.
    # Matches the Phase 2 waybar pattern (systemd.enable = true, no exec-once).
    # Do NOT also add `caelestia` to Hyprland exec-once -- that double-starts it.
    systemd = {
      enable = true;
      target = "graphical-session.target";
    };

    # Put the `caelestia` CLI on PATH so Hyprland keybinds can invoke it
    # (launcher toggle, clipboard, screenshot, record). Handled in Plan 02.
    cli.enable = true;

    # Behaviour-only settings. Fonts/colors deferred -- caelestia ships
    # Rubik + CaskaydiaCove by default; font pivot to Cinzel/Noto Serif
    # Display is deferred out of Phase 2.1 (see roadmap Phase 3 rework).
    settings = {
      bar.status.showBattery = true;
      paths.wallpaperDir = "~/Pictures/wallpapers";
    };
  };
}
```
