Add the three foundation user-facing tools of the desktop shell: Waybar status bar, Rofi app launcher, and Kitty terminal. These are declared as home-manager modules so they live in the user session and inherit Stylix theming automatically.

Output: Three new home-manager module files plus an updated aggregator. On rebuild, a Waybar bar appears at the top, `rofi -show drun` launches apps, and Kitty opens as the default `$terminal`.
</objective>

<task type="auto">
  <name>Task 1: Create waybar.nix home-manager module</name>
  <files>modules/home/waybar.nix</files>
  <action>
Create `modules/home/waybar.nix` as a new file. It MUST NOT contain a `style` attribute or any colors/fonts -- Stylix owns theming via `stylix.targets.waybar` which is auto-enabled.

Required structure (from RESEARCH.md "Waybar (home-manager)" code example):

```nix
{ ... }:
{
  programs.waybar = {
    enable = true;
    systemd.enable = true;
    settings.mainBar = {
      layer = "top";
      position = "top";
      height = 30;
      spacing = 4;
      modules-left = [ "hyprland/workspaces" "hyprland/window" ];
      modules-center = [ "clock" ];
      modules-right = [ "pulseaudio" "network" "battery" "tray" ];

      "hyprland/workspaces" = { format = "{id}"; };
      clock = {
        format = "{:%Y-%m-%d  %H:%M}";
        tooltip-format = "<big>{:%Y %B}</big>\n<tt><small>{calendar}</small></tt>";
      };
      pulseaudio = {
        format = "{volume}% {icon}";
        format-muted = "MUTE";
        format-icons.default = [ "" "" "" ];
        on-click = "wpctl set-mute @DEFAULT_AUDIO_SINK@ toggle";
      };
      network = {
        format-wifi = "{essid} {signalStrength}%";
        format-ethernet = "eth";
        format-disconnected = "--";
        on-click = "nm-connection-editor";
      };
      battery = {
        format = "{capacity}% {icon}";
        format-icons = [ "" "" "" "" "" ];
        states = { warning = 30; critical = 15; };
      };
      tray.spacing = 8;
    };
    # NO style block -- Stylix owns waybar theming via stylix.targets.waybar.
  };
}
```

Rationale for choices (do NOT deviate):
- `systemd.enable = true`: lets Hyprland's graphical-session target manage the service instead of needing `exec-once = waybar` in Hyprland config.
- Tray module is included NOW so nm-applet (added in 02-04) and cliphist status icons appear without a second edit.
- `pulseaudio` module name is correct even though we're running PipeWire -- it speaks the PulseAudio protocol.
- Do NOT add `style = ...` or any `color`/`background`/`font` setting. Stylix will overwrite and/or collide.
  </action>
  <verify>
`test -f modules/home/waybar.nix && grep -q 'hyprland/workspaces' modules/home/waybar.nix && ! grep -qE '^\s*style\s*=' modules/home/waybar.nix`
  </verify>
  <done>
File exists at modules/home/waybar.nix, contains `programs.waybar.enable = true`, declares modules-left/center/right with the modules listed above, and contains NO `style` attribute and NO color values.
  </done>

**modules/home/kitty.nix** (from RESEARCH.md "Kitty (home-manager)"):

```nix
{ ... }:
{

    enable = true;
    settings = {
      enable_audio_bell = false;

```

Rationale:
- `pkgs.rofi-wayland` (not `pkgs.rofi`) is mandatory -- research confirms nixpkgs still ships rofi-wayland as the blessed Wayland variant. Using plain `rofi` causes broken rendering on Hyprland.
- No `font` setting anywhere: Stylix injects JetBrains Mono globally. If we override we fight Stylix.
