  <name>Task 3: Extend modules/home/hyprland.nix with keybinds and exec-once</name>
  <files>modules/home/hyprland.nix</files>
  <action>
Extend `modules/home/hyprland.nix` to wire every Phase 2 tool into Hyprland's settings. The file currently has `wayland.windowManager.hyprland.settings = { monitor, $mainMod, $terminal, general, decoration, input, bind, bindm }`.

Changes to make (RESEARCH.md "Hyprland extension" code example + Pattern 4 + Pattern 5):

**1. Add `exec-once` list inside `settings`** (new attribute, place after `input` and before `bind`):

```nix
exec-once = [
  "nm-applet --indicator"   # waybar tray WiFi/ethernet icon
  "hyprpolkitagent"         # graphical polkit prompts for mount/WiFi admin
];
```

Rationale: Hyprland does not implement XDG autostart, so `programs.nm-applet.enable = true` alone does not launch the applet -- we MUST exec it here (RESEARCH.md Pattern 4 + Pitfall 2). `hyprpolkitagent` ensures Thunar USB mount and nm-connection-editor get GUI sudo prompts (RESEARCH.md Pitfall 5 + Open Question 2 resolution).

**2. Extend the existing `bind` list** -- APPEND (do not replace) these entries to the existing `bind = [ ... ]`:

```
"$mainMod, SPACE, exec, rofi -show drun"
"$mainMod, L, exec, loginctl lock-session"
"$mainMod, V, exec, cliphist list | rofi -dmenu -display-columns 2 | cliphist decode | wl-copy"
", Print, exec, grimblast copy area"
"SHIFT, Print, exec, grimblast copy screen"
```

Keep all existing binds from Phase 1 (Super+T, Super+Q, Super+M/exit, Super+V/togglefloating, Super+F/fullscreen, workspace switching, window focus).

**IMPORTANT conflict:** Phase 1's bind list has `"$mainMod, V, togglefloating,"`. This collides with the new clipboard bind. RESOLVE by changing the Phase 1 entry to Super+Shift+V for floating toggle:

- Replace `"$mainMod, V, togglefloating,"` with `"$mainMod SHIFT, V, togglefloating,"`

Super+V now = clipboard history (KEY-07, explicit requirement), and Super+Shift+V = togglefloating (preserved at the cost of a modifier).

**3. Add `bindel` list** (for keys that repeat on hold and work while input is grabbed) inside `settings`, after `bind`:

```nix
bindel = [
  ", XF86AudioRaiseVolume, exec, wpctl set-volume -l 1.0 @DEFAULT_AUDIO_SINK@ 5%+"
  ", XF86AudioLowerVolume, exec, wpctl set-volume @DEFAULT_AUDIO_SINK@ 5%-"
  ", XF86AudioMute,        exec, wpctl set-mute @DEFAULT_AUDIO_SINK@ toggle"
  ", XF86MonBrightnessUp,  exec, brightnessctl s +5%"
  ", XF86MonBrightnessDown,exec, brightnessctl s 5%-"
];
```

Volume cap at 1.0 prevents accidental distortion (the `-l 1.0` flag to wpctl).

**4. Add `bindl` list** (keys that work even when the session is locked -- needed for media keys on lock screen) after `bindel`:

```nix
bindl = [
  ", XF86AudioPlay,  exec, playerctl play-pause"
  ", XF86AudioNext,  exec, playerctl next"
  ", XF86AudioPrev,  exec, playerctl previous"
];
```

**Do NOT:**
- Touch the existing `monitor`, `general`, `decoration`, `input`, `bindm` blocks.
- Use the old `bind = ... ,e,...` single-letter flag form -- it is replaced by distinct `bindel`/`bindl`/`bindr` variants.
- Add `exec-once = waybar`: `programs.waybar.systemd.enable = true` (Plan 02-01) handles waybar startup via systemd user service.
- Add `exec-once = hyprpaper`: `services.hyprpaper.enable = true` (Plan 02-03) handles startup via user systemd.
