<task type="auto">
  <name>Task 1: Create flake.nix with all inputs and nixosConfigurations output</name>
  <files>flake.nix</files>
  <action>
Create `flake.nix` at the repository root with the following exact structure:

```nix
{
  description = "NervOS - Evangelion-themed NixOS";

  inputs = {
    nixpkgs.url = "github:NixOS/nixpkgs/nixos-unstable";

    home-manager = {
      url = "github:nix-community/home-manager";
      inputs.nixpkgs.follows = "nixpkgs";
    };

    stylix = {
      url = "github:nix-community/stylix";
      inputs.nixpkgs.follows = "nixpkgs";
    };
  };

  outputs = { self, nixpkgs, home-manager, stylix, ... }:
  let
    system = "x86_64-linux";
  in {
    nixosConfigurations.nervos = nixpkgs.lib.nixosSystem {
      inherit system;
      specialArgs = { inherit self; };
      modules = [
        ./hosts/nervos
        # Hyprland: wired in Plan 01-02
        # Stylix + home-manager: wired in Plan 01-03
      ];
    };
  };
}
```

Critical details:
- Use `specialArgs = { inherit self; }` -- NOT `_module.args`. Using `_module.args` causes infinite recursion when values are referenced in `imports`.
- All three inputs declared now even though Stylix and home-manager modules are commented out -- this ensures `nix flake lock` generates a complete `flake.lock` with all dependencies pinned.
- Both `home-manager` and `stylix` MUST have `inputs.nixpkgs.follows = "nixpkgs"` to share a single nixpkgs instance.
- Do NOT add a Hyprland flake input -- Hyprland is upstreamed to nixpkgs and will be used via `programs.hyprland.enable`.
  </action>
  <verify>
Run `cd /path/to/VanguardOS && nix flake check` (if on NixOS) or visually confirm:
1. Three inputs declared: nixpkgs, home-manager, stylix
2. Both home-manager and stylix have `inputs.nixpkgs.follows = "nixpkgs"`
3. `specialArgs` used (not `_module.args`)
4. `nixosConfigurations.nervos` output defined
  </verify>
  <done>flake.nix exists with 3 inputs, all using follows, specialArgs pattern, and nixosConfigurations.nervos output</done>
