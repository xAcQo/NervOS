Update `flake.nix` to import the Stylix and home-manager NixOS modules into `nixosConfigurations.nervos`. Replace the entire outputs block to produce this final `flake.nix`:

```nix
{
  description = "NervOS - Evangelion-themed NixOS";

  inputs = {
    nixpkgs.url = "github:NixOS/nixpkgs/nixos-unstable";

    home-manager = {
      url = "github:nix-community/home-manager";
      inputs.nixpkgs.follows = "nixpkgs";
    };

    stylix = {
      url = "github:nix-community/stylix";
      inputs.nixpkgs.follows = "nixpkgs";
    };
  };

  outputs = { self, nixpkgs, home-manager, stylix, ... }:
  let
    system = "x86_64-linux";
  in {
    nixosConfigurations.nervos = nixpkgs.lib.nixosSystem {
      inherit system;
      specialArgs = { inherit self; };
      modules = [
        ./hosts/nervos
        stylix.nixosModules.stylix
        home-manager.nixosModules.home-manager
        {
          home-manager.useGlobalPkgs = true;
          home-manager.useUserPackages = true;
          home-manager.users.pilot = import ./modules/home;
        }
      ];
    };
  };
}
```

Critical details:
- Module order matters only for `mkDefault`/`mkForce` priority. This order (host -> stylix -> home-manager -> HM inline config) is the community-standard pattern.
- `home-manager.useGlobalPkgs = true` -- HM uses the system nixpkgs (no separate HM pkgs instance).
- `home-manager.useUserPackages = true` -- HM installs user packages to `/etc/profiles/per-user/pilot` instead of `~/.nix-profile`.
- The inline module `{ home-manager.users.pilot = import ./modules/home; }` points at the directory created in Plan 01-02. NixOS will look for `./modules/home/default.nix`.
- Do NOT import `stylix.homeModules.stylix` anywhere. Stylix auto-integrates into home-manager when both are NixOS modules. Manual import causes double-definition errors on every Stylix option.
- Keep all three `follows = "nixpkgs"` declarations. Removing any causes multiple nixpkgs instances.
  </action>
  <verify>
1. `flake.nix` modules list contains `stylix.nixosModules.stylix` and `home-manager.nixosModules.home-manager`
2. `flake.nix` contains `home-manager.useGlobalPkgs = true` and `home-manager.useUserPackages = true`
3. `flake.nix` contains `home-manager.users.pilot = import ./modules/home`
4. All three inputs still declare `inputs.nixpkgs.follows = "nixpkgs"`
5. No reference to `stylix.homeModules.stylix` anywhere
6. `nix flake lock` (if Nix available) succeeds; `flake.lock` contains entries for nixpkgs, home-manager, stylix
  </verify>

Critical details:
- Use `pkgs.nerd-fonts.jetbrains-mono` -- NOT `pkgs.nerdfonts` or `nerdfonts.override`. The monolithic `nerdfonts` package was split in nixpkgs ~2024 and the old syntax is now a hard error.
- `stylix.image` is REQUIRED -- Stylix fails to evaluate without a wallpaper. Path is `../../assets/wallpaper.jpg` relative to `modules/nixos/stylix.nix`, resolving to the repo-root `assets/wallpaper.jpg`.
- DejaVu Sans is a deliberate Phase 1 placeholder. Phase 3 swaps to Matisse EB (or closest open alternative). The Stylix declaration makes swap trivial.
- `polarity = "dark"` applies dark theme preference to GTK and apps that respect it.
- Do NOT set fonts or palette at home-manager level -- Stylix auto-propagates to HM.
