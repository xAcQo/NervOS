    **modules/home/waybar.nix:**

    Add `{ lib, ... }:` to the function signature (currently `{ ... }:`), then append a `style` attribute using `lib.mkAfter`. The full module becomes:

    ```nix
    { lib, ... }:
    {
      programs.waybar = {
        enable = true;
        systemd.enable = true;
        settings.mainBar = {
          # ... existing settings unchanged ...
        };
        style = lib.mkAfter ''
          /* NERV Command: mechanical title compression */
          #window label {
            transform: scaleX(0.82);
            transform-origin: left center;
            letter-spacing: 0.08em;
            text-transform: uppercase;
            font-weight: bold;
          }

          /* Active workspace: NERV orange accent + bottom bar */
          #workspaces button.active {
            color: @base09;
            border-bottom: 2px solid @base09;
            background: transparent;
          }

          /* Clock: wire cyan */
          #clock {
            color: @base0C;
            font-weight: bold;
          }

          /* Battery critical: alert red */
          #battery.critical {
            color: @base08;
          }
        '';
      };
    }
    ```

    IMPORTANT: Use `#window label` (not `#window`) as the CSS selector so scaleX(0.82) applies to the text content node, not the outer container widget — applying transform to the container clips the widget bounds without visually compressing the text.

    IMPORTANT: `lib.mkAfter` appends after Stylix's generated CSS. Do NOT use `lib.mkForce` here — that would replace Stylix's CSS and lose the base palette styling. The `@base09`, `@base0C`, `@base08` CSS custom properties are provided by Stylix's waybar target (which remains enabled — only `addCss` default behavior is supplemented, not disabled).

    **modules/home/hyprland.nix:**

    Add `{ lib, ... }:` to the function signature (currently `{ config, pkgs, ... }:`). Then extend the `settings` block with three additional sections. Merge these into the existing settings — do NOT replace the existing bind/bindel/bindl/exec-once blocks, only add/override the visual settings:

    ```nix
    general = {
      gaps_in = 4;          # Replaces Phase 1 value of 5
      gaps_out = 8;         # Replaces Phase 1 value of 10
      border_size = 2;      # Unchanged
      layout = "dwindle";   # Unchanged
      "col.active_border" = lib.mkForce "rgba(ff9830ff)";    # NERV orange — must be quoted (contains dot)
      "col.inactive_border" = lib.mkForce "rgba(1a1a1aff)";  # Phase 1 dark bg
    };

    decoration = {
      rounding = 0;          # MAGI straight-edge — replaces Phase 1 rounding = 8
      shadow.enabled = false; # Hyprland v0.41+ syntax
    };

    animations = {
      enabled = true;
      bezier = [ "nervOut, 0.16, 1, 0.3, 1" ];  # Sharp ease-out — institutional/mechanical feel
      animation = [
        "windows, 1, 3, nervOut, popin 85%"
        "windowsIn, 1, 3, nervOut, popin 85%"
        "windowsOut, 1, 2, nervOut, popin 85%"
        "fade, 1, 3, nervOut"
        "workspaces, 1, 4, nervOut, slide"
        "border, 1, 5, nervOut"
      ];
    };
    ```

    SYNTAX NOTES (critical — these are known pitfalls from research):
    - `"col.active_border"` MUST be quoted in Nix attribute names (the dot makes it a non-identifier)
    - `lib.mkForce` is required because Stylix sets border colors via its hyprland target; without mkForce Stylix wins and NERV orange won't appear
    - `shadow.enabled` (not `drop_shadow`) — Hyprland v0.41+ renamed this; use the new form
    - `bezier` and `animation` MUST use list syntax `[ "..." ]` — a single string will cause Hyprland parse errors
    - The `general` block in settings merges with the existing `general` block — Nix attribute set merge means the two `general = { ... }` entries in the settings attrset will conflict; instead, write ONE `general` block with all keys (gaps_in, gaps_out, border_size, layout, col.active_border, col.inactive_border)

    **modules/home/mako.nix:**

    mako's `border-radius` is a layout/shape property (not a color or font), so it is safe to set directly alongside Stylix's color theming — no conflict. Add these two settings to the existing `services.mako.settings` (or `services.mako`) block:

    ```nix
    border-radius = 0;   # MAGI straight-edge — overrides any default rounding
    border-size = 1;     # Clean single-pixel NERV border line
    ```

    Do NOT set any color, font, or background properties in mako.nix — Stylix owns those. Only the geometry/shape settings are set here.

    **modules/home/rofi.nix — Matisse EB font guarantee:**

    After the rebuild from Plan 01 is complete, check whether Matisse EB reached Rofi via Stylix inheritance:

    ```bash
    grep -i matisse ~/.config/rofi/*.rasi 2>/dev/null || grep -i matisse ~/.cache/stylix/rofi* 2>/dev/null
    ```

    If Matisse EB IS found in Rofi's generated config — no action needed. Stylix inheritance is working correctly.

    If Matisse EB is NOT found in Rofi's generated config — add the following to `modules/home/rofi.nix`. This is a shape/display override (not a color/theme override) and is safe alongside Stylix theming:

    ```nix
    programs.rofi.font = "MatissePro-EB 12";
    ```

    This explicit font declaration overrides Stylix's font inheritance for Rofi only, ensuring Matisse EB appears in the launcher header/prompt regardless of whether Stylix's Rofi target propagates the sansSerif font correctly.
