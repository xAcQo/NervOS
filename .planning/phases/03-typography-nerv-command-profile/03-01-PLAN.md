      meta.description = "Matisse EB display font (Fontworks) — NERV Command display/title typeface";
    }

<task type="auto">
  <name>Task 2: Wire matisse-eb via local callPackage in stylix.nix</name>
  <files>
    modules/nixos/stylix.nix
  </files>
  <action>
    **modules/nixos/stylix.nix changes:**

    Use a local `let`-binding with `pkgs.callPackage` directly inside stylix.nix. This is the canonical pattern — no specialArgs or flake.nix overlay needed for the font package.

    ```nix
    # modules/nixos/stylix.nix
    { config, pkgs, ... }:
    let
      matisse-eb = pkgs.callPackage ../../pkgs/matisse-eb {};
    in {
      # ... existing stylix config ...

      stylix.fonts.sansSerif = {
        package = matisse-eb;
        name = "MatissePro-EB";  # PostScript name — verify with fc-list after rebuild
      };

      fonts.packages = [ matisse-eb ];
    }
    ```

    Specific changes to make to the existing stylix.nix:

    1. Change the function signature from `{ config, pkgs, ... }:` to `{ config, pkgs, ... }:` — no change needed here since `pkgs` is already in scope.

    2. Add a `let ... in` block at the top of the file (before the `{` that opens the module attrset) with: `matisse-eb = pkgs.callPackage ../../pkgs/matisse-eb {};`

    3. Replace the `sansSerif` font block (DejaVu placeholder) with:
    ```nix
    sansSerif = {
      package = matisse-eb;
      name = "MatissePro-EB";
    };
    ```

    4. Keep the serif entry pointing to DejaVu (serif is not a Phase 3 target — Matisse EB is display/sansSerif only).

    5. Add `stylix.targets.hyprpaper.enable = false;` inside the `stylix` block to prevent Stylix from overwriting wallpaper.nix's hyprpaper settings. This is REQUIRED — without it Stylix generates a conflicting hyprpaper config.

    6. Add `fonts.packages = [ matisse-eb ];` at the NixOS top-level (outside the `stylix` block, at module root) so fontconfig indexes the OTF for GTK apps and fc-list queries.

    The color palette in stylix.nix is already the correct NERV Command base16 palette from Phase 1 — do NOT change it. The inline comments noting "Phase 3 tunes this" can be updated to "Phase 3 complete — NERV Command palette finalized" but the hex values stay identical.

    NOTE: flake.nix does NOT need any specialArgs entry for matisse-eb. The callPackage is self-contained inside stylix.nix. Do not add matisse-eb to flake.nix specialArgs.

    **JetBrains Mono nerd font blank-glyph contingency:** The current stylix.nix already declares `pkgs.nerd-fonts.jetbrains-mono` / `"JetBrainsMono Nerd Font Mono"`. Do NOT change this. The blank-glyph bug (nixpkgs #366979) may or may not affect the target machine — the verification step in Plan 02 will surface it. If it does manifest, the fix is to change the name to `"JetBrains Mono"` (non-Nerd variant) or swap to `pkgs.nerd-fonts.hack` / `"Hack Nerd Font Mono"`.
  </action>
