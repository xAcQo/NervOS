<task type="auto">
  <name>Task 2: Expose matisse-eb via flake.nix overlay and wire into stylix.nix</name>
  <files>
    flake.nix
    modules/nixos/stylix.nix
  </files>
  <action>
    **flake.nix changes:**

    Add a `pkgsFor` let-binding with an overlay that exposes matisse-eb via `callPackage`, then thread it into the NixOS configuration via `specialArgs`:

    ```nix
    outputs = { self, nixpkgs, home-manager, stylix, ... }:
    let
      system = "x86_64-linux";
      pkgs = import nixpkgs { inherit system; };
      matisse-eb = pkgs.callPackage ./pkgs/matisse-eb {};
    in {
      nixosConfigurations.nervos = nixpkgs.lib.nixosSystem {
        inherit system;
        specialArgs = { inherit self matisse-eb; };
        modules = [
          ./hosts/nervos
          stylix.nixosModules.stylix
          home-manager.nixosModules.home-manager
          {
            home-manager.useGlobalPkgs = true;
            home-manager.useUserPackages = true;
            home-manager.users.pilot = import ./modules/home;
          }
        ];
      };
    };
    ```

    This makes `matisse-eb` available as a top-level argument in any NixOS module that declares it in its function signature.

    **modules/nixos/stylix.nix changes:**

    1. Add `matisse-eb` to the function signature: `{ config, pkgs, matisse-eb, ... }:`

    2. Replace the `sansSerif` font block (DejaVu placeholder) with Matisse EB:
    ```nix
    sansSerif = {
      package = matisse-eb;
      name = "MatissePro-EB";  # PostScript name — verify with fc-list after rebuild
    };
    ```

    3. Keep the serif entry pointing to DejaVu (serif is not a Phase 3 target — Matisse EB is display/sansSerif only).

    4. Add `stylix.targets.hyprpaper.enable = false;` inside the `stylix` block to prevent Stylix from overwriting wallpaper.nix's hyprpaper settings. This is REQUIRED — without it Stylix generates a conflicting hyprpaper config.

    5. Add `fonts.packages = [ matisse-eb ];` at the NixOS top-level (outside the `stylix` block, at module root) so fontconfig indexes the OTF for GTK apps and fc-list queries.

    The color palette in stylix.nix is already the correct NERV Command base16 palette from Phase 1 — do NOT change it. The inline comments noting "Phase 3 tunes this" can be updated to "Phase 3 complete — NERV Command palette finalized" but the hex values stay identical.

    **JetBrains Mono nerd font blank-glyph contingency:** The current stylix.nix already declares `pkgs.nerd-fonts.jetbrains-mono` / `"JetBrainsMono Nerd Font Mono"`. Do NOT change this. The blank-glyph bug (nixpkgs #366979) may or may not affect the target machine — the verification step in Plan 02 will surface it. If it does manifest, the fix is to change the name to `"JetBrains Mono"` (non-Nerd variant) or swap to `pkgs.nerd-fonts.hack` / `"Hack Nerd Font Mono"`.
  </action>
